# v0.11 Deployment Chain Package

v0.11 packages the project for a practical GitHub -> Azure -> remote MCP deployment chain.

## Added

- `.github/workflows/azure-container-app.yml`
- `infra/main.bicep`
- `scripts/smoke-remote-mcp.mjs`
- `scripts/print-azure-deploy-help.mjs`
- `docs/deploy-chain-runbook.md`

## Deployment stance

This release does not assume every consultant installs the MCP locally. It supports a more scalable pattern:

```text
GitHub repository
  -> GitHub Actions
  -> Azure Container Apps
  -> Remote MCP endpoint
  -> ChatGPT / MCP client
```

## Security stance

- `/mcp` requires a bearer token by default.
- `/health` exposes only basic service metadata.
- SharePoint configuration is explicit and should start with a restricted site/library.
- Secrets are held in GitHub Actions and Azure Container Apps secrets, not committed to the repository.
