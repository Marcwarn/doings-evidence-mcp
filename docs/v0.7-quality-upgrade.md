# v0.7 Quality Upgrade

v0.7 turns Doings Evidence MCP from a broad evidence scanner into a more explicit critical review assistant.

## New capabilities

### Claim decomposition

`critique_claim` now decomposes compound claims before assessment. This helps avoid treating a sentence like:

> Autonomous teams make organizations more agile and reduce the need for middle management.

as one claim. v0.7 separates multiple outcomes and shows how each subclaim differs in testability and level of analysis.

### Level-of-analysis detection

Claims and evidence are tagged heuristically as:

- individual
- team
- unit
- organization
- ecosystem
- mixed_or_unclear

The server flags when a claim is organizational but the evidence is mostly individual- or team-level.

### Context-fit scoring

The server now assesses whether the retrieved evidence matches the provided context. Signals include geography, sector, organization size, work mode and dependency structure.

Possible values:

- high
- medium
- low
- unknown

Context fit is intentionally conservative. A relevant paper is not automatically a transferable paper.

### Study-type classification

`rate_evidence_quality` now returns a study-type profile, including:

- meta_analysis
- systematic_review
- field_experiment
- experiment
- longitudinal_study
- cross_sectional_survey
- qualitative_case_study
- mixed_methods
- theory_or_conceptual
- practitioner_report
- unknown

This reduces the risk of treating conceptual papers, case studies and causal empirical designs as equivalent.

### Full-text passage extraction

When open-access full text is available, v0.7 attempts to extract passages that look like:

- supporting passages
- contradicting / cautionary passages
- method passages
- limitations passages

These are heuristic excerpts from extracted text, not verified page-perfect citations.

### Red-team mode

`critique_claim` accepts:

```json
{
  "redTeamMode": true
}
```

When enabled, the output includes:

- what would make the claim false
- skeptical reviewer objections
- alternative explanations
- recommended stress test

For SharePoint document audits, use:

```json
{
  "validateHighRiskClaims": true,
  "validationRedTeamMode": true
}
```

## Important limitation

v0.7 is still an exploratory evidence scan, not a systematic literature review. It improves rigor by checking decomposition, method type, context fit and passage-level signals, but it does not implement a formal PRISMA workflow, inclusion/exclusion coding or human double review.
