# v0.8 User-facing critique layer

v0.8 changes the product from a research tool into a more useful day-to-day thinking partner for Doings consultants.

## Design premise

Users rarely write clean claims. They write rough questions, draft sentences, half-finished pitch language and client hypotheses. The MCP should therefore detect what the user is trying to do before choosing how much research detail to return.

## New tools

### `critique_org_text`

Best general entry point. It detects intent, extracts a primary claim, runs the evidence critique engine, flags consulting-language risk and returns a narrative answer.

### `can_we_say_this`

Fast entry point for checking whether a sentence is safe enough to use.

## Modes

- `quick_check`: short practical verdict
- `rewrite_safely`: safer wording for pitch/RFP/report text
- `red_team`: skeptical challenge
- `evidence_brief`: fuller evidence-oriented answer
- `auto`: infer mode from the user input

## Intent detection examples

Input: “Kan vi säga att autonoma team gör organisationer snabbare?”

Selected mode: `quick_check`

Input: “Skriv om detta så att det blir mer forskningshederligt.”

Selected mode: `rewrite_safely`

Input: “Var är detta svagt?”

Selected mode: `red_team`

Input: “Vad säger forskningen om självstyrande team?”

Selected mode: `evidence_brief`

## Consulting-language risk

The risk detector flags common overclaim patterns such as:

- unlocks potential/agility/value
- drives transformation/agility/alignment
- reduces complexity/overhead/management
- empowers/future-proofs
- always/never/guarantees
- best practice/proven model

These are not automatically wrong, but they require a clearer mechanism, level of analysis and boundary conditions.

## Product principle

The user-facing layer should help the user answer:

- Can we say this?
- What should we avoid implying?
- What is the safer formulation?
- Where is the reasoning weakest?
- What would a skeptical reviewer or client challenge?

It should not bury the user in raw research metadata unless `includeRawCritique=true` or the selected mode is `evidence_brief`.
