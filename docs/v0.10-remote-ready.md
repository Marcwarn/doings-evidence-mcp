# v0.10 Remote-ready MCP

v0.10 makes Doings Evidence MCP deployable as a hosted service while preserving local STDIO mode.

## New capabilities

- `src/mcpServer.ts`: shared MCP server factory.
- `src/server.ts`: local STDIO entrypoint.
- `src/httpServer.ts`: hosted Streamable HTTP entrypoint.
- `/health` endpoint.
- `/mcp` Streamable HTTP endpoint.
- Bearer-token protection for hosted mode.
- Dockerfile and `.dockerignore`.
- Azure deployment notes.
- Local and remote example client config.
- Smoke-health script.

## Scripts

```bash
npm run dev          # local STDIO dev
npm run build        # TypeScript build
npm run start        # local STDIO from dist
npm run dev:http     # hosted HTTP dev
npm run start:http   # hosted HTTP from dist
npm run smoke:health # calls /health
```

## Pilot recommendation

Use local STDIO for development and power users.
Use hosted HTTP for a small Doings pilot once Azure deployment and SharePoint scope are approved.
